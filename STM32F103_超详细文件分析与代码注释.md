## 📘 第一部分：如何从文件名获取信息（文件命名规则）

### 1.1 文件扩展名告诉你什么？

| 扩展名 | 含义 | 作用 | 示例 |
|--------|------|------|------|
| `.h` | Header File（头文件） | 声明函数、宏定义、结构体、**被其他`.c`文件包含** | `stm32f10x_gpio.h` |
| `.c` | C Source File（C源文件） | 实现具体功能、**被编译器编译** | `stm32f10x_gpio.c` |
| `.s` | Assembly Source File（汇编源文件） | 启动代码、中断向量表、**最先执行的代码** | `startup_stm32f10x_hd.s` |
| `.uvprojx` | Keil Project File（Keil工程文件） | 双击打开Keil5工程 | `Template.uvprojx` |
| `.hex` | Intel HEX File（十六进制文件） | 可烧录到STM32的固件文件 | `UAS.hex` |
| `.md` | Markdown File（标记语言文件） | 项目说明文档、**用文本编辑器打开** | `README.md` |
| `.xlsx` | Excel File（Excel文件） | 资源分配表、**用Excel打开** | `STM32资源使用情况.xlsx` |
| `.jpg/.png` | Image File（图片文件） | 硬件连接图、**用图片查看器打开** | `WFR07.jpg` |

### 1.2 文件命名规律（STM32标准外设库）

#### 规律1：`stm32f10x_xxx.h` / `stm32f10x_xxx.c`
- **含义**：STM32F10x系列的标准外设库文件
- **xxx**：外设名称（如gpio、rcc、usart、tim等）
- **示例**：
  - `stm32f10x_gpio.h` → GPIO（通用输入输出）外设的**头文件**
  - `stm32f10x_gpio.c` → GPIO外设的**源文件**（具体实现）
  - `stm32f10x_usart.h` → USART（串口）外设的头文件
  - `stm32f10x_tim.h` → TIM（定时器）外设的头文件

#### 规律2：`startup_stm32f10x_xxx.s`
- **含义**：STM32F10x的启动文件（汇编语言）
- **xxx**：芯片容量型号
  - `ld` → Low Density（小容量：16KB~32KB Flash）
  - `md` → Medium Density（中容量：64KB~128KB Flash）
  - `hd` → High Density（大容量：256KB~512KB Flash）← **本项目使用**
  - `xl` → Extra Large Density（超大容量：768KB~1024KB Flash）
  - `cl` → Connectivity Line（互联型：STM32F105/107）
  - `vl` → Value Line（超值型：STM32F100）
- **示例**：
  - `startup_stm32f10x_hd.s` → **大容量STM32F103ZET6的启动文件**（本项目使用）

#### 规律3：用户模块命名（`User/模块名/模块名.c/h`）
- **含义**：用户自己编写的功能模块
- **命名方式**：`模块名/模块名.c`（文件夹名和文件名相同）
- **示例**：
  - `User/GPS/GPS.c` → GPS模块的功能实现
  - `User/GPS/GPS.h` → GPS模块的函数声明、宏定义
  - `User/PID/PID.c` → PID控制模块的功能实现
  - `User/Motor/Motor.c` → 电机控制模块的功能实现

### 1.3 文件夹命名规则

| 文件夹路径 | 含义 | 里面有什么 |
|-----------|------|-----------|
| `Libraries/CMSIS/` | Cortex微控制器软件接口标准 | 核心支持、设备支持 |
| `Libraries/STM32F10x_StdPeriph_Driver/` | STM32F10x标准外设驱动库 | 所有外设的`.h`和`.c`文件 |
| `User/` | 用户代码目录 | **你自己写的所有功能模块** |
| `User/GPS/` | GPS模块代码 | GPS初始化、数据解析 |
| `User/PID/` | PID控制模块代码 | PID算法实现 |
| `User/Motor/` | 电机控制模块代码 | 电机初始化、速度控制 |
| `Project/` | Keil工程文件 | `.uvprojx`工程文件、编译配置 |
| `Output/` | 编译输出目录 | `.hex`固件文件 |
| `obj/` | 编译中间文件 | `.hex`文件（另一个输出位置） |

#### 📁 `Libraries/CMSIS/CM3/DeviceSupport/ST/STM32F10x/`

##### 📄 `stm32f10x.h`
- **从文件名获取的信息**：
  - `stm32f10x` → STM32F10x系列
  - `.h` → 头文件
- **作用**：
  - **最重要的头文件**！定义了STM32所有寄存器的地址映射
  - 例如：`GPIOA`、`GPIOB`、`RCC`、`USART1`等所有外设的寄存器结构体
  - 所有其他`.c`文件都会包含这个头文件
- **你需要做什么**：

##### 📄 `system_stm32f10x.c` / `system_stm32f10x.h`
- **从文件名获取的信息**：
  - `system_stm32f10x` → STM32F10x系统初始化
  - `.c` / `.h` → 源文件和头文件
- **作用**：
  - **系统时钟配置**（最重要的文件之一！）
  - `SystemInit()`函数：设置系统时钟（内部8MHz → 外部8MHz晶振 → PLL倍频到72MHz）
  - `SystemCoreClock`变量：记录当前系统时钟频率（72MHz）
- **你需要做什么**：
  - **理解`SystemInit()`函数**：它把系统时钟配置为72MHz
  - 如果你的外部晶振不是8MHz，需要修改这个文件

#### 📁 `Libraries/CMSIS/CM3/DeviceSupport/ST/STM32F10x/startup/`

##### 📄 `startup_stm32f10x_hd.s`（汇编文件）
- **从文件名获取的信息**：
  - `startup` → 启动文件

- **你需要做什么**：
  - **理解启动流程**：上电 → 执行`Reset_Handler` → 调用`SystemInit()` → 调用`main()`
  - 如果需要修改堆栈大小，可以修改这个文件

### 3.2 `User/initialize.c` — 初始化函数（系统初始化核心）

```c
/* 
============================================================
文件：User/initialize.c
功能：初始化所有外设
说明：这个函数在main()函数开始时被调用，初始化所有需要用到的外设
============================================================
*/

# include "inclusion.h"  // 包含用户模块的头文件

/* 
============================================================
宏定义：串口波特率
说明：
  1. USART1_BAUDRATE - 上位机通信波特率（921600bps，很高！）
  2. GPS_BAUDRATE_CONFIG - GPS模块配置波特率（115200bps）
  3. 为什么USART1波特率这么高？因为上位机需要接收大量姿态数据，需要高速通信
  4. 为什么GPS波特率配置为115200？因为GPS模块默认是9600bps，我们需要通过UBX协议把它配置为115200bps
============================================================
*/
# define USART1_BAUDRATE 921600   // 上位机通信波特率（很高！）
# define GPS_BAUDRATE_CONFIG 115200 // GPS模块配置波特率

/* 
============================================================
函数：void Initialize(void)
功能：初始化所有外设（核心初始化函数！）
参数：无
返回：无
说明：
  这个函数被main()函数调用，初始化所有需要用到的外设。
  初始化顺序很重要！有些外设依赖于其他外设已经初始化。
============================================================
*/
void Initialize(void)
{
  /*==========第一步：NVIC中断分组配置===========*/
  /* 
  --- NVIC中断分组 ---
  NVIC（Nested Vectored Interrupt Controller，嵌套向量中断控制器）
  STM32的中断优先级分为：抢占优先级和子优先级
  这里设置为NVIC_PriorityGroup_2：
  - 抢占优先级：2位（0~3，共4级）
  - 子优先级：2位（0~3，共4级）
  
  抢占优先级高的中断可以打断抢占优先级低的中断（中断嵌套）
  抢占优先级相同的情况下，子优先级高的先执行
  */
  NVIC_PriorityGroupConfig(NVIC_PriorityGroup_2); // 设置NVIC中断分组2:2位
  
  /*==========第二步：初始化延时函数===========*/
  /* 
  --- 初始化延时函数 ---
  delay_init()函数在User/Delay/delay.c中定义
  它使用SysTick定时器实现精确延时
  SystemCoreClock是系统时钟频率（72MHz）
  */
  delay_init(SystemCoreClock);
  
  /*==========第三步：初始化系统节拍定时器===========*/
  /* 
  --- 初始化TIM5定时器（系统节拍定时器）---
  TIM5_SYSTEM_PULSE_Init()函数在User/LoopSequence/Loop.c中定义
  TIM5定时器产生5ms中断，用于任务调度
  在TIM5的中断服务函数中，设置各个任务的执行标志位
  */
  TIM5_SYSTEM_PULSE_Init();
  
  /*==========第四步：初始化GPIO引脚===========*/
  /* 
  --- 初始化GPIO引脚 ---
  GPIO_Pin_Init()函数初始化所有用到的GPIO引脚
  例如：
  - LED引脚（PE5、PE6）：用于指示系统状态
  - 电机控制引脚（PB8、PB9等）：用于控制电机正转、反转、刹车
  - 舵机控制引脚（PA0等）：用于PWM输出
  - 等等
  */
  GPIO_Pin_Init();
  
  /*==========第五步：初始化I2C===========*/
  /* 
  --- 初始化I2C ---
  IIC_Init()函数初始化I2C外设（或软件模拟I2C）
  MPU6050和HMC5883都通过I2C接口与STM32通信
  */
  IIC_Init();
  
  /*==========第六步：初始化环形缓冲区===========*/
  /* 
  --- 初始化环形缓冲区 ---
  ringBuff_Init()和ringBuff_IMU_Init()函数初始化环形缓冲区
  环形缓冲区用于：
  - GPS数据缓冲（GPS模块发送数据很快，MCU处理不过来，需要缓冲）
  - IMU数据缓冲（MPU6050数据缓冲）
  */
  ringBuff_Init();
  ringBuff_IMU_Init();
  
  /*==========第七步：配置电机引脚===========*/
  /* 
  --- 配置电机引脚 ---
  Motor_IN_Pin_Init()函数初始化电机控制引脚
  初始化后，电机处于刹车状态（防止意外转动）
  */
  Motor_IN_Pin_Init();
  Motor_Brake(ALL_WHEEL);  // 电机刹车（锁定所有轮子）
  
  /*==========第八步：初始化舵机===========*/
  /* 
  --- 初始化舵机 ---
  Servo_Set_Angle(0)函数设置舵机角度为0°（居中）
  延时700ms，等待舵机转动到指定角度
  */
  Servo_Set_Angle(0);  // 舵机转到0°（居中）
  delay_ms(700); // 延时，等待外设全部启动
  
  /*==========第九步：初始化USART1（与上位机通信）===========*/
  /* 
  --- 初始化USART1 ---
  usart_init()函数初始化USART1外设
  参数：
  - 第一个参数：波特率（921600bps）
  - 第二个参数：是否使能DMA接收（0 = 不使能）
  - 第三个参数：是否使能DMA发送（0 = 不使能）
  
  注意：在初始化阶段，不使能DMA，因为DMA配置需要在所有外设初始化完成后再开启
  */
  usart_init(USART1_BAUDRATE,0,0);  // 初始化USART1时不使能DMA
  
  /*==========第十步：配置无线串口（未使用）===========*/
  /* 
  --- 配置无线串口 ---
  WirelessPort_Online = 0：不使用无线串口
  如果使用无线串口，需要初始化无线模块
  */
  WirelessPort_Online = 0;  // 不使用无线串口
  
  /*==========第十一步：配置GPS模块===========*/
  /* 
  --- 配置GPS模块（UBlox M8）---
  GPSDATA_Init()函数初始化GPS模块
  配置步骤：
  1. 初始化USART3（与GPS通信），波特率设置为115200bps
  2. 发送UBX协议命令，配置GPS输出速率为10Hz
  3. 发送UBX协议命令，配置GPS只输出必要的信息（PVT数据）
  4. 发送UBX协议命令，配置GPS动态模型（车载模型或海上模型）
  
  为什么要在初始化阶段配置GPS？
  因为GPS模块默认输出所有信息（包括NMEA协议的各种语句），数据量很大，
  MCU处理不过来。所以我们需要通过UBX协议配置GPS，只输出必要的信息。
  */
  GPSDATA_Init();  // 初始化GPS数据解析
  
  //usart3_init(GPS_BAUDRATE_CONFIG);  // 这行被注释掉了
  //delay_ms(5);
  //UBX_CFG_USART(115200);  // 这行被注释掉了
  
  delay_ms(5);
  usart3_init(115200);  // GPS串口波特率设置（115200bps）
  delay_ms(5);
  // 修改GPS波特率（通过UBX协议配置GPS模块的串口波特率为115200bps）
  UBX_CFG_USART(GPS_BAUDRATE_CONFIG);
  
  delay_ms(5);
  usart3_init(GPS_BAUDRATE_CONFIG);  // 配置成功，则GPS波特率被修改
  delay_ms(5);
  UBX_CFG_RATE();  // 配置GPS输出速率（10Hz）
  delay_ms(5);
  UBX_CFG_DATAOUT();  // 配置GPS输出信息（只输出PVT数据）
  delay_ms(5);
  UBX_CFG_MODEL();  // 配置GPS动态模型（车载模型或海上模型）
  
  /*==========第十二步：配置磁力计（HMC5883）===========*/
  /* 
  --- 配置磁力计 ---
  verifyCompass()函数检测磁力计是否存在（通过I2C读取芯片ID）
  如果存在，则调用cfgCompassReg()函数配置磁力计寄存器
  */
  if(verifyCompass()) 
  {
    printf("\nCompass Online!\n");  // 打印提示信息（通过串口发送到上位机）
    cfgCompassReg();  // 配置磁力计寄存器
  }
  
  /*==========第十三步：初始化MPU6050===========*/
  /* 
  --- 初始化MPU6050 ---
  MPU6050_Init()函数初始化MPU6050（加速度计 + 陀螺仪）
  verifyMPU6050()函数检测MPU6050是否存在（通过I2C读取芯片ID）
  */
  MPU6050_Init();
  if(verifyMPU6050()) printf("\nMPU6050 Online!\n");
  
  /*==========第十四步：执行传感器校准程序===========*/
  /* 
  --- 校准传感器 ---
  Calibrate_Gyro()函数校准陀螺仪（消除零点漂移）
  Calibrate_Mag()函数校准磁力计（消除硬铁干扰和软铁干扰）
  
  校准过程：
  1. 陀螺仪校准：将设备静止放置，采集100组数据，计算平均值，作为零点漂移
  2. 磁力计校准：将设备在空中做"8"字形运动，采集各个方向的磁场数据，计算校准参数
  */
  Calibrate_Gyro(100);  // 校准陀螺仪（采集100组数据）
  Calibrate_Mag();       // 校准磁力计
  
  /*==========第十五步：获取初始姿态===========*/
  /* 
  --- 获取初始姿态 ---
  1. receiveComData()函数读取MPU6050数据（加速度计和陀螺仪）
  2. dataFrom6050()函数处理MPU6050数据（转换为物理量）
  3. filter_IMU_data(0)函数滤波IMU数据（去除噪声）
  4. 计算初始姿态角（Roll、Pitch、Yaw）
  5. 将初始姿态转换为四元数（因为Madgwick算法使用四元数表示姿态）
  */
  receiveComData();  // 读取MPU6050数据
  dataFrom6050();    // 处理MPU6050数据
  filter_IMU_data(0);  // 滤波IMU数据
  
  // 计算初始姿态角（使用加速度计数据计算Roll和Pitch，使用磁力计数据计算Yaw）
  theta = atan2(pAcceleration.channelY, pAcceleration.channelZ);  // Pitch角
  psi = atan2(pAcceleration.channelX, pAcceleration.channelZ);   // Roll角
  fhi = 3.1415926 - atan2((float)pCompass_Main.channelY, (float)pCompass_Main.channelX);  // Yaw角
  fhi -= Local_mag_bias;  // 减去当地磁偏角
  
  // 将初始姿态角转换为四元数（因为Madgwick算法使用四元数表示姿态）
  SEq_4 = cos(fhi / 2) * cos(theta / 2) * cos(psi / 2) + sin(fhi / 2) * sin(theta / 2) * sin(psi / 2);
  SEq_1 = sin(fhi / 2) * cos(theta / 2) * cos(psi / 2) - cos(fhi / 2) * sin(theta / 2) * sin(psi / 2);
  SEq_2 = cos(fhi / 2) * sin(theta / 2) * cos(psi / 2) + sin(fhi / 2) * cos(theta / 2) * sin(psi / 2);
  SEq_3 = cos(fhi / 2) * cos(theta / 2) * sin(psi / 2) - sin(fhi / 2) * sin(theta / 2) * cos(psi / 2);
  
  Convert_Quaternion_To_Euler();  // 将四元数转换为欧拉角（验证转换是否正确）
  
  /*==========第十六步：启动USART1的DMA===========*/
  /* 
  --- 启动USART1的DMA ---
  在初始化阶段，不使能DMA（因为DMA配置需要在所有外设初始化完成后再开启）
  现在所有外设都初始化完成了，可以开启DMA了
  DMA可以提高数据传输效率，不占用CPU
  */
  usart_init(USART1_BAUDRATE, DMA_RX, DMA_TX);  // 初始化USART1，并使能DMA
  
  /*==========第十七步：启动遥控器信号捕获===========*/
  /* 
  --- 启动遥控器信号捕获 ---
  TIM3_Capture_Init()函数初始化TIM3定时器的输入捕获功能
  用于测量遥控器发出的PWM信号的脉宽，解析遥控器各个通道的值
  */
  TIM3_Capture_Init();
  
  /*==========第十八步：配置看门狗===========*/
  /* 
  --- 配置看门狗 ---
  WDG_Initialize(7, 240)函数初始化独立看门狗（IWDG）
  参数：
  - 第一个参数：预分频器（7 = 256分频）
  - 第二个参数：重装载值（240）
  
  超时时间计算：
  LSI时钟频率 = 40kHz（典型值）
  预分频器 = 256
  重装载值 = 240
  超时时间 = (240 * 256) / 40000 = 1.536秒
  
  如果在1.536秒内没有喂狗（调用WDG_Feed()函数），看门狗会复位系统
  */
  WDG_Initialize(7, 240);  // 配置看门狗超时时间为1.536秒
  
  /*==========第十九步：启动看门狗===========*/
  /* 
  --- 启动看门狗 ---
  WDG_Start()函数启动独立看门狗
  启动后，必须在超时时间内喂狗，否则系统会复位
  */
  WDG_Start();  // 最后启动看门狗
}
```

### 3.3 `User/LoopSequence/Loop.c` — 主循环任务调度（核心架构！）

> **说明**：这个文件实现了**类操作系统式任务调度**，是项目的核心架构。我会详细解释任务调度原理和有限状态机。

```c
/* 
============================================================
文件：User/LoopSequence/Loop.c
功能：主循环任务调度（类操作系统式框架）
说明：
  这个文件实现了类操作系统式任务调度，任务按不同频率被调用：
  - 200Hz（5ms）：最高优先级任务
  - 100Hz（10ms）：高频任务
  - 50Hz（20ms）：姿态解算
  - 10Hz（100ms）：传感器数据读取
  - 5Hz（200ms）：与上位机通信
  - 1Hz（1000ms）：系统状态监控、喂狗
  
  优点：
  1. 提高系统灵活性
  2. 降低系统耦合性
  3. 类似无人机飞控厂商（如Crazyflie）的设计
============================================================
*/

# define LOOP 1  // 定义LOOP宏（用于条件编译）
# include "..\CONFIGURATION.h"  // 包含配置文件

/* 
============================================================
任务调度原理说明：
  
  系统节拍：5ms（由TIM5定时器产生中断）
  
  在TIM5的中断服务函数（TIM5_IRQHandler）中，根据system_pulse计数器的值，
  设置各个任务的执行标志位：
  - system_pulse每递增1次，表示过了5ms
  - system_pulse % 1 == 0：每5ms执行一次（200Hz）
  - system_pulse % 2 == 0：每10ms执行一次（100Hz）
  - system_pulse % 4 == 0：每20ms执行一次（50Hz）
  - system_pulse % 20 == 0：每100ms执行一次（10Hz）
  - system_pulse % 40 == 0：每200ms执行一次（5Hz）
  - system_pulse % 200 == 0：每1000ms执行一次（1Hz）
  
  在Loop()函数中，检查各个任务的执行标志位，如果为1，则执行相应任务
============================================================
*/

/* 
============================================================
函数：void Loop(void)
功能：主循环任务调度（核心函数！）
参数：无
返回：无（死循环中被反复调用）
说明：
  这个函数在main()函数的while(1)死循环中被反复调用
  它检查各个任务的执行标志位，如果为1，则执行相应任务
  
  任务执行时间限制（考虑到中断等突发事件）：
  - 200Hz任务：T5 <= 5ms
  - 100Hz任务：T10 - T5 = 5ms
  - 50Hz任务：T20 - T10 = 15ms
  - 10Hz任务：T100 - T20 = 80ms
  - 1Hz任务：T1000 - T100 = 900ms
============================================================
*/
void Loop(void)
{
  /*--------1Hz级任务（最低优先级，每1000ms执行一次）---------*/
  if(Mission.Hz1)  // 如果1Hz任务标志位为1
  {
    feed_dog();  // 喂狗（重置看门狗计数器，防止系统复位）
    
    /* 
    --- 有限状态机 ---
    根据当前系统状态（current_state），执行不同的操作
    系统状态有：
    - STATE_INITIALIZE：初始化状态（不会在这里出现，因为初始化已经完成）
    - STATE_NORMALLY_RUNNING：程序正常运行（自主导航）状态
    - STATE_COLLECTING_POINTS：GPS采点状态
    - STATE_MANUAL_CTRL：手动遥控状态
    - STATE_DEBUGGING：程序测试状态
    */
    switch(current_state)
    {
      /* Limited State Machine: 程序正常运行状态 */
      case STATE_NORMALLY_RUNNING:
      {
        if(RaspiOfflineIndicator > 0) RaspiOfflineIndicator--;  // 上位机离线计数器递减
        break;
      }
      case STATE_MANUAL_CTRL:    // 手动遥控状态
      case STATE_COLLECTING_POINTS:  // GPS采点状态
      {
        if(MadgwickGainAutoDecend >= MadgwickGainDecendTime) 
          MadgwickGainAutoDecend = MadgwickGainDecendTime;  // 限制增益自动下降时间
        else 
          MadgwickGainAutoDecend++;  // 用于使Madgwick增益自动缩小（让算法快速收敛）
        
        LED_BLINK();  // LED闪烁（指示系统正在运行）
        break;
      }				
      /* Limited State Machine: 调试状态 */
      case STATE_DEBUGGING:
      {
        PrintAttitudeData();  // 打印姿态数据（用于调试）
        break;
      }
    }
    
    Mission.Hz1 = 0;  // 清除1Hz任务标志位
  }
  
  /*--------5Hz级任务（每200ms执行一次）---------*/
  if(Mission.Hz5)  // 如果5Hz任务标志位为1
  {
    switch(current_state)  // 根据当前系统状态执行不同操作
    {
      /* Limited State Machine: 程序正常运行状态 */
      case STATE_NORMALLY_RUNNING:
      {
        #if !USE_USART1_COMM_DMA_RX  // 如果不使用DMA接收
          if(resolve_frame_from_Host_computer())  // 解析上位机发送的数据帧
          {
            MISSION_LED_BLINK();  // 任务LED闪烁（指示接收到上位机数据）
            RaspiOfflineIndicator = MotorLockTimeAfterRaspiOffline;  // 重置上位机离线计数器
          }
        #endif
        break;
      }
      case STATE_MANUAL_CTRL:    // 手动遥控状态
      case STATE_COLLECTING_POINTS:  // GPS采点状态
      {
        synthesis_frame();  // 向上位机发送当前姿态数据
        break;
      }
      /* Limited State Machine: 调试状态 */
      case STATE_DEBUGGING:
      {
        TIM3_4channel_process();  // 处理遥控器4个通道的数据
        printf("\nTim3-CH1:%d\n", CHANNEL_HIGHOUTPUT[0]);  // 打印通道1的值（用于调试）
        break;
      }
    }
    
    Mission.Hz5 = 0;  // 清除5Hz任务标志位
  }
  
  /*--------10Hz级任务（每100ms执行一次）---------*/
  if(Mission.Hz10)  // 如果10Hz任务标志位为1
  {
    TIM3_4channel_process();  // 处理遥控器4个通道的数据
    
    /* 
    --- 有限状态机驱动器以10Hz的频率运行 ---
    Analyze_Controller_Msg()函数分析遥控器消息，切换系统状态
    遥控器通道0用于切换系统状态：
    - 通道0 <= 1200：GPS采点状态
    - 1200 < 通道0 <= 1800：自主导航状态
    - 通道0 > 1800：手动遥控状态
    */
    CHANGE_STATE_TO(Analyze_Controller_Msg());  // 切换系统状态
    
    /* 
    --- 根据当前系统状态执行不同操作 ---
    */
    switch(current_state)
    {
      /* Limited State Machine: 程序正常运行状态 */
      case STATE_NORMALLY_RUNNING:
      {
        /* 
        --- 自主导航 ---
        如果上位机在线、GPS数据有效、电机已使能，
        则执行路径规划（自主导航）
        */
        if(RaspiOfflineIndicator && SignalFromGPS() && Motor_Enabled) 
        {
          Execute_Planned_Path(yaw, routeToGo.yaw, routeToGo.distance, routeToGo.control_status, 10);
        }
        else 
        {
          Servo_Set_Angle(0);  // 舵机转到0°（居中）
          Motor_Brake(ALL);       // 电机刹车（停止）
        }		
        break;
      }
      case STATE_COLLECTING_POINTS:  // GPS采点状态
      case STATE_MANUAL_CTRL:        // 手动遥控状态
      {
        #if !USE_USART3_GPS_DMA_RX  // 如果不使用DMA接收GPS数据
          pvtFromGPS();  // 解析GPS数据
        #endif
        
        receiveComData();  // 读取MPU6050数据
        dataFrom6050();   // 处理MPU6050数据
        filter_IMU_data(10);  // 滤波IMU数据（10Hz滤波）
        break;
      }
      /* Limited State Machine: 调试状态 */
      case STATE_DEBUGGING:
      {
        print_IMU_data();  // 打印IMU数据（用于调试）
        break;
      }
    }
    
    Mission.Hz10 = 0;  // 清除10Hz任务标志位
  }
  
  /*--------50Hz级任务（每20ms执行一次，姿态解算）---------*/
  if(Mission.Hz50)  // 如果50Hz任务标志位为1
  {
    switch(current_state)  // 根据当前系统状态执行不同操作
    {
      case STATE_MANUAL_CTRL:    // 手动遥控状态
      case STATE_COLLECTING_POINTS:  // GPS采点状态
      case STATE_NORMALLY_RUNNING:  // 程序正常运行状态
      {
        /* 
        --- Madgwick梯度下降姿态解算算法 ---
        MadgwickAHRSupdate()函数实现Madgwick梯度下降姿态解算算法
        输入：
        - 陀螺仪数据（pAngleSpeed.channelX, pAngleSpeed.channelY, pAngleSpeed.channelZ）
        - 加速度计数据（pAcceleration.channelX, pAcceleration.channelY, pAcceleration.channelZ）
        - 磁力计数据（pCompass_Main.channelX, pCompass_Main.channelY, pCompass_Main.channelZ）
        - 增益参数（(MadgwickGainDecendTime - MadgwickGainAutoDecend) + BETA）
        
        输出：
        - 姿态四元数（SEq_1, SEq_2, SEq_3, SEq_4）
        
        为什么选择Madgwick算法？
        1. 低性能要求：仅需10Hz采样速度即可收敛
        2. 弥补卡尔曼滤波缺点：卡尔曼滤波吃性能
        3. 适合STM32F1：72MHz主频无法同时运行姿态解算和控制（使用Madgwick可以）
        */
        MadgwickAHRSupdate(
          pAngleSpeed.channelX, pAngleSpeed.channelY, pAngleSpeed.channelZ,  // 陀螺仪数据
          pAcceleration.channelX, pAcceleration.channelY, pAcceleration.channelZ,  // 加速度计数据
          pCompass_Main.channelX, pCompass_Main.channelY, pCompass_Main.channelZ,  // 磁力计数据
          (MadgwickGainDecendTime - MadgwickGainAutoDecend) + BETA  // 增益参数
        );
        
        Convert_Quaternion_To_Euler();  // 将四元数转换为欧拉角（Roll、Pitch、Yaw）
        break;
      }
      
      /* Limited State Machine: 调试状态 */
      case STATE_DEBUGGING:
      {
        break;
      }
    }
    
    Mission.Hz50 = 0;  // 清除50Hz任务标志位
  }
  
  /*--------100Hz级任务（每10ms执行一次）---------*/
  if(Mission.Hz100)  // 如果100Hz任务标志位为1
  {
    Mission.Hz100 = 0;  // 清除100Hz任务标志位（目前没有任务）
  }
  
  /*--------200Hz级任务（每5ms执行一次，最高优先级）---------*/
  if(Mission.Hz200)  // 如果200Hz任务标志位为1
  {
    Mission.Hz200 = 0;  // 清除200Hz任务标志位（目前没有任务）
  }
}

/* 
============================================================
函数：void TIM5_SYSTEM_PULSE_Init(void)
功能：初始化TIM5定时器（系统节拍定时器）
参数：无
返回：无
说明：
  TIM5定时器产生5ms中断，用于任务调度
  在TIM5的中断服务函数中，设置各个任务的执行标志位
============================================================
*/
void TIM5_SYSTEM_PULSE_Init(void)
{
  TIM_TimeBaseInitTypeDef TIM_TimeBaseStructure;  // TIM时基初始化结构体
  NVIC_InitTypeDef NVIC_InitStructure;            // NVIC初始化结构体
  
  RCC_APB1PeriphClockCmd(RCC_APB1Periph_TIM5, ENABLE);  // 使能TIM5时钟（APB1总线）
  
  TIM_DeInit(TIM5);  // 复位TIM5定时器
  
  TIM_TimeBaseStructure.TIM_CounterMode = TIM_CounterMode_Up;  // 向上计数模式
  TIM_TimeBaseStructure.TIM_ClockDivision = TIM_CKD_DIV1;     // 时钟分频：不分频
  TIM_TimeBaseStructure.TIM_Period = 100;                    // 自动重装载值：100
  TIM_TimeBaseStructure.TIM_Prescaler = (7200) / APB1DIV - 1;  // 预分频器：7200分频（0.05ms中断一次）
  
  TIM_TimeBaseInit(TIM5, &TIM_TimeBaseStructure);  // 初始化TIM5时基
  
  TIM_ITConfig(TIM5, TIM_IT_Update, ENABLE);  // 使能TIM5更新中断
  
  // 中断优先级NVIC设置
  NVIC_InitStructure.NVIC_IRQChannel = TIM5_IRQn;  // TIM5中断
  NVIC_InitStructure.NVIC_IRQChannelPreemptionPriority = 0;  // 抢占优先级：0级（最高）
  NVIC_InitStructure.NVIC_IRQChannelSubPriority = 0;        // 子优先级：0级
  NVIC_InitStructure.NVIC_IRQChannelCmd = ENABLE;              // IRQ通道使能
  NVIC_Init(&NVIC_InitStructure);  // 初始化NVIC寄存器
  
  TIM_Cmd(TIM5, ENABLE);  // 使能TIM5
  TIM_ARRPreloadConfig(TIM5, ENABLE);  // 自动重装载使能
  TIM_Cmd(TIM5, ENABLE);  // 再次使能TIM5（确保使能）
  
  system_pulse = 1;  // 系统节拍计数器初始化为1
  
  /* 清除所有任务标志位 */
  Mission.Hz1 = 0;
  Mission.Hz10 = 0;
  Mission.Hz50 = 0;
  Mission.Hz100 = 0;
  Mission.Hz200 = 0;
}

/* 
============================================================
函数：void TIM5_IRQHandler(void)
功能：TIM5中断服务函数（任务调度驱动器）
参数：无
返回：无
说明：
  这个函数在TIM5定时器溢出时被调用（每5ms调用一次）
  它根据system_pulse计数器的值，设置各个任务的执行标志位
============================================================
*/
void TIM5_IRQHandler(void)  // TIM5中断
{
  if (TIM_GetITStatus(TIM5, TIM_IT_Update) != RESET)  // 检查TIM5更新中断是否发生
  {
    /* 
    --- 任务调度systick驱动器 ---
    system_pulse每递增1次，表示过了5ms
    根据system_pulse的值，设置各个任务的执行标志位
    */
    system_pulse++;  // 系统节拍计数器递增
    
    if((system_pulse + 1) % (1) == 0)   { Mission.Hz200 = 1; }  // 200Hz任务（5ms）
    if((system_pulse + 1) % (2) == 0)   { Mission.Hz100 = 1; }  // 100Hz任务（10ms）
    if((system_pulse + 1) % (4) == 0)   { Mission.Hz50  = 1; }  // 50Hz任务（20ms）
    if((system_pulse + 1) % (20) == 0)  { Mission.Hz10  = 1; }  // 10Hz任务（100ms）
    if((system_pulse + 1) % (40) == 0)  { Mission.Hz5   = 1; }  // 5Hz任务（200ms）
    if((system_pulse + 1) % (200) == 0)                   // 1Hz任务（1000ms）
    {
      Mission.Hz1 = 1;
      system_pulse = 0;  // 重置系统节拍计数器
    }
    
    TIM_ClearITPendingBit(TIM5, TIM_IT_Update);  // 清除TIM5更新中断标志位
  }
}

/*

函数：u8 Analyze_Controller_Msg(void)
功能：分析遥控器消息，切换系统状态

返回：新的系统状态（STATE_MANUAL_CTRL / STATE_COLLECTING_POINTS / STATE_NORMALLY_RUNNING / STATE_DEBUGGING）

  */
  if(CHANNEL_HIGHOUTPUT[0] <= bottom_limit)

  }
  else if(CHANNEL_HIGHOUTPUT[0] <= upper_limit)
  {
    Status = STATE_NORMALLY_RUNNING;  // 自主导航状态
  }
  else
  {
    Status = STATE_MANUAL_CTRL;  // 手动遥控状态
  }
  
  /* 
  --- 根据遥控器通道1切换电机使能 ---
  */
  if(CHANNEL_HIGHOUTPUT[1] <= bottom_limit)
  {
    Motor_Enabled = FALSE;  // 电机不使能
  }
  else if(CHANNEL_HIGHOUTPUT[1] <= upper_limit)
  {
    Motor_Enabled = FALSE;  // 电机不使能
  }
  else
  {
    Motor_Enabled = TRUE;  // 电机使能
  }
  
  /* 
  --- 手动遥控模式下，控制舵机和电机 ---
  */
  #if USE_BRUSHED_MOTOR  // 如果使用有刷电机
    if(Motor_Enabled)  // 如果电机已使能
    {
      if(Status == STATE_MANUAL_CTRL || Status == STATE_COLLECTING_POINTS)
      {
        MISSION_LED_BLINK();  // 任务LED闪烁
        
        /* 
        --- 控制电机速度 ---
        如果遥控器通道3的值 >= 1700 或 <= 1300，则控制电机转动
        否则，电机刹车

        if(CHANNEL_HIGHOUTPUT[3] >= 1700 || CHANNEL_HIGHOUTPUT[3] <= 1300)
          Motor_Function(ALL, Minimize_Greatest_Error_Increment_PID(
            Read_Speed_left_wheel(), 
            Read_Speed_right_wheel(), 
            (0.85 * MOTOR_MAX_SPEED * speed)

        else
          Motor_Brake(ALL);  // 电机刹车

        如果角度 >= 2° 或 <= -2°，则控制舵机转动
        否则，舵机居中（0°）
        */
        if(angle >= 2 || angle <= -2)
          Servo_Set_Angle(angle);  // 设置舵机角度
        else
          Servo_Set_Angle(0);  // 舵机居中
      }
    }
    else  // 如果电机未使能
    {
      Servo_Set_Angle(0);  // 舵机居中
      Motor_Brake(ALL);       // 电机刹车
    }
  #endif
  
  return Status;  // 返回新的系统状态
```

## 📊 第四部分：核心算法详解

### 4.1 Madgwick梯度下降姿态解算算法

> **说明**：这是项目的核心算法，用于融合加速度计、陀螺仪、磁力计的数据，计算出姿态四元数。

**算法原理**：
1. **梯度下降法**：在机器学习中广泛应用，用于优化问题
2. **目标**：使解算姿态与真实姿态之间的误差逐步下降
3. **优点**：对迭代次数、传感器采样频率要求很低（仅需10Hz）
4. **适合STM32F1**：72MHz主频无法同时运行卡尔曼滤波和姿态解算，使用Madgwick可以

**算法步骤**：
1. 初始化四元数（使用加速度计和磁力计数据计算初始姿态）
2. 在每次迭代中，计算目标函数的梯度
3. 沿梯度下降方向更新四元数
4. 归一化四元数（保证四元数为单位四元数）
5. 将四元数转换为欧拉角（Roll、Pitch、Yaw）

**代码位置**：`User/DATAFUSION/DATAFUSION.c`

---

### 4.2 PID控制算法

> **说明**：PID是最常用的控制算法，用于使电机转速达到目标值。

**算法原理**：
1. **P（比例）**：当前误差 × Kp → 快速响应
2. **I（积分）**：误差累积 × Ki → 消除稳态误差
3. **D（微分）**：误差变化率 × Kd → 抑制震荡

**增量式PID**：
- 计算的是控制量的增量（而不是控制量的绝对值）
- 优点：避免积分饱和，适合电机控制

**代码位置**：`User/PID/PID.c`
